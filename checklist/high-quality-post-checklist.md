# High-Quality Post Checklist

## Before Writing

- 이 글이 답하는 질문이 하나로 좁혀져 있는가
- 독자가 실제로 겪는 실패 상황을 한 문장으로 말할 수 있는가
- 이 글의 핵심 명제를 한 줄로 적을 수 있는가
- 본문에 넣을 판단 기준 3개 이상이 이미 정리돼 있는가
- 실제 예시 2개 이상을 넣을 수 있는가

## During Writing

- 첫 문단이 독자의 실패 상황을 바로 잡는가
- 범위가 빨리 드러나는가
- 글에서 가장 중요한 섹션 하나가 분명한가
- 핵심 섹션이 나머지보다 더 자세하고 더 흥미롭게 전개되는가
- 각 주요 섹션에 판단 기준이나 예시가 있는가
- 설명만 있고 기준이 없는 문단이 남아 있지 않은가
- 본문 안에 복사 가능한 요소가 최소 1개 있는가
- 내부 링크 안내 문장이 본문 밀도를 대신하고 있지 않은가
- 이미지 없이도 글이 기본적으로 읽히는가
- 추가 이미지는 새로운 설명을 맡을 때만 넣고 있는가

## Before Review

- 제목이 문제와 보상을 같이 말하는가
- description이 클릭 이유를 주는가
- 핵심 명제가 글 전체를 묶고 있는가
- 글이 설명 페이지처럼 평평하지 않고, 강약이 있는가
- 예시가 추상 설명을 실제 장면으로 내려주고 있는가
- 이미지가 글 길이 부족을 가리는 용도로 쓰이지 않았는가
- hero 이미지가 광고형 mockup, 카드 보드, 텍스트 많은 UI처럼 보이지 않는가
- 이미지 안에 직접적인 단어 라벨(`Before`, `TITLE`, `locale URL`, `Platform A`)이 들어가 있지 않은가
- KO/EN 문장 안에 작업 중 섞인 어색한 혼합 표현이 남아 있지 않은가
- 마지막 문단이 지금 할 첫 행동을 말하는가

## Review Gate

- `문제`가 선명한가
- `핵심 섹션`이 실제로 가장 강한가
- `판단 기준`이 최소 3개 있는가
- `실제 예시`가 최소 2개 있는가
- `복사 가능한 요소`가 최소 1개 있는가
- `내부 링크 없이 읽어도` 글이 성립하는가
- `이미지 없이도` 글이 기본적으로 성립하는가
- 추가 이미지가 있다면 `새 설명`을 맡고 있는가
- `001`은 핵심 대비가 바로 읽히고, `002`는 본문 구조 설명에만 집중하는가
- 이미지는 글자 없이도 구조가 읽히는가
- `결론`이 다음 행동을 분명히 제시하는가

## Fail Signs

- 맞는 말은 많은데 남는 기준이 없다
- 글이 다른 글 안내 문장에 기대고 있다
- 개념 설명은 있는데 예시가 없다
- 읽고 나서 독자가 무엇을 먼저 해야 하는지 모른다
- 섹션별 차이가 약하고 반복 설명이 많다
- 모든 섹션 길이와 톤이 비슷해서, 포스팅이 아니라 설명 페이지처럼 보인다
- 이미지 수는 많은데 텍스트 자체의 밀도는 약하다
- 추가 이미지가 같은 말을 다른 방식으로 한 번 더 반복할 뿐이다
- hero가 예쁜 UI 샘플처럼 보이지만 글 핵심 진단을 직접 보여주지 못한다
- 이미지를 라벨 텍스트를 읽지 않으면 거의 이해할 수 없다
- 한 언어 본문 안에 작업 중 생긴 다른 언어 표현이 남아 있다
