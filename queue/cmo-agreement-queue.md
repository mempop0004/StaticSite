# CMO Agreement Queue

이 문서는 Codex가 혼자 확정하면 안 되는 항목을 모아두는 합의 대기 큐다.

## Status Labels

- `open`: 아직 합의되지 않음
- `agreed`: 사용자와 합의 완료
- `blocked`: 정보 부족으로 보류

## Agreed Items

### 1. 프로젝트의 1차 수익 모델

- status: `agreed`
- agreed: 1차 수익 모델은 `애드센스`다
- note: 쿠팡 같은 링크형 제휴는 추후 후보로 다룬다
- need agreement: 예

### 2. 1차 타깃 고객

- status: `agreed`
- agreed: 독자를 먼저 정하지 않는다
- agreed: 주제를 먼저 정하고, 각 주제에 최적화된 독자층을 붙인다
- agreed: 블로그 하나는 주제 하나만 가진다
- note: 타깃 고객은 전역 선정보다 블로그 주제 단위 매칭으로 관리한다
- need agreement: 예

### 3. 블로그의 핵심 CTA

- status: `agreed`
- agreed: 현재 단계의 핵심 CTA는 판매 CTA가 아니라 `광고 노출에 유리한 사용자 행동`이다
- note: 여기서 CTA는 "사용자가 다음으로 하길 원하는 행동"을 뜻한다
- note: 애드센스 우선 단계에서는 클릭 유도 문구보다 `추가 글 읽기`, `페이지 체류`, `관련 글 이동`이 더 중요하다
- need agreement: 예

### 4. 운영 역할 분리 수준

- status: `agreed`
- agreed: 역할은 `나`, `너(관리)`, `너의 보조(실행)` 구조로 운영한다
- note: 실무 문서에서는 `Owner`, `Manager`, `Worker`로 표기한다
- need agreement: 예
