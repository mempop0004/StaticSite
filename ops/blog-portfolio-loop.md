# Blog Portfolio Loop

이 문서는 `블로그 1개 = 주제 1개` 원칙으로 여러 블로그를 운영하고, 성과로 남길 것과 버릴 것을 결정하는 운영 루프다.

## Core Principle

- 확정: 독자를 먼저 고정하지 않는다
- 확정: 블로그 주제를 최대한 많이 정한다
- 확정: 블로그 하나는 주제 하나만 가진다
- 확정: 각 블로그 주제마다 그 주제에 최적화된 독자층을 붙인다
- 확정: 블로그별 자동화를 실행해 `28일간` 운영해 본 뒤 결과로 판단한다
- 확정: 하루 운영 최소치는 `2개 블로그`다
- 확정: 하루 운영 상한은 `8시간 안에 감당 가능한 개수`다
- 확정: 효율이 안 나오는 블로그는 버린다

## Operating Flow

1. 블로그 주제 후보를 많이 모은다
2. 각 주제에 맞는 독자층을 붙인다
3. 주제별 블로그와 자동화 실행 단위를 만든다
4. 블로그별로 `28일간` 운영한다
5. 결과를 보고 우선순위를 재정렬한다
6. 유지, 보류, 폐기를 결정한다

## Blog Unit

블로그 하나는 아래 항목으로 관리한다.

- 블로그명
- 핵심 주제
- 주제 유형
- 예상 독자층
- 검색 잔존 수요
- 자동화 가능 여부
- 운영 시작일
- 운영 기간
- 결과 메모
- 결정 상태

## Topic Types

- 기술 문제 해결형
- 툴 비교형
- 자동화 실전형
- 필요하면 이후 확장

## Decision Status

- `candidate`: 아직 실행 전
- `running`: 현재 자동화 운영 중
- `review`: 운영 기간 종료, 평가 대기
- `keep`: 계속 운영
- `pause`: 잠시 보류
- `drop`: 폐기

## Evaluation Rule

- 확정: 블로그 평가는 느낌이 아니라 결과로 한다
- 가정: 현재 기본 평가 기준은 검색 유입, 체류, 관련 글 이동, 광고 수익 기여다
- 확정: 운영 실험 기간은 `28일`이다
- 확정: 하루 운영 최소치는 `2개 블로그`다
- 확정: 하루 운영 상한은 고정 숫자가 아니라 `8시간 안에 감당 가능한 개수`다

## Management Rule

- `Manager`는 블로그 포트폴리오를 정리하고 우선순위를 조정한다
- `Worker`는 블로그별 자동화 실행과 결과 기록을 담당한다
- `Owner`는 keep, pause, drop 판단의 최종 승인자다
