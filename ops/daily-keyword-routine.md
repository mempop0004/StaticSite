# Daily Keyword Routine

## Goal

- 루틴 시작 시 각 블로그에서 오늘 쓸 수 있는 키워드를 뽑고, 최소 1개 글을 발행 가능한 상태까지 밀어붙인다.
- 기본 완료 규칙은 `ops/content-completion-system.md`를 따른다.

## Daily Rule

1. 각 블로그에서 키워드 후보를 3개까지 뽑는다.
2. 후보는 아래 4축으로 본다.
   - 수요: 실제 검색할 만한 문제인가
   - 즉시 작성 가능성: 오늘 바로 실전 글로 쓸 수 있는가
   - 내부 연결성: 기존 글, 유닛 페이지, 후속 글과 자연스럽게 이어지는가
   - 확장성: 같은 클러스터 안에서 2~3개 파생 글이 더 나오는가
3. 각 블로그에서 순위를 이렇게 나눈다.
   - 1순위: 오늘 바로 쓰는 글
   - 2순위: 1순위가 막히면 바로 대체할 글
   - 3순위: 후속 글 또는 같은 클러스터 확장 글
4. 당일 메인 블로그 1개를 정한다.
5. 기본 목표는 `최소 1개 작성`이다.
6. 여유가 있으면 같은 클러스터에서 최대 3개까지 쓴다.

## Selection Rule

- 넓은 주제보다 문제 해결형, 비교형, 체크리스트형을 우선한다.
- 오늘 예시, 화면, 실제 체크포인트를 넣을 수 없는 주제는 1순위로 올리지 않는다.
- 기존 글과 내부 링크가 붙지 않는 키워드는 우선순위를 낮춘다.
- 제목만 강한 주제보다 후속 글로 확장 가능한 주제를 우선한다.

## Previous Post Impact Window

- 고정 일수는 없다. 글 유형에 따라 다르다.
- 운영 기준은 아래 3단계로 본다.
  - `D+7`: 초기 반응 확인
  - `D+14`: 제목, 설명, 내부 링크 수정 후 재판단
  - `D+28`: 유지 가치 판단

## Interpretation Rule

- 뉴스성, 트렌드성 글은 빨리 죽을 수 있다.
- 문제 해결형, 운영형 글은 늦게 쌓일 수 있다.
- 이 프로젝트의 주력은 운영형 글이므로 `7일 반응`보다 `28일 유지`를 더 중시한다.

## Keep / Expand / Stop Rule

- `D+7` 안에 반응이 있으면 후속 글 후보를 우선 작성한다.
- `D+14`에 제목, 설명, 내부 링크를 다시 점검한다.
- `D+28`까지 의미 있는 신호가 없으면 그 주제 확장은 멈춘다.

## Daily Output

- 블로그별 키워드 3순위
- 오늘 메인 글 1개
- 여유가 있으면 후속 1~2개
- 이전 글의 `D+7`, `D+14`, `D+28` 체크 메모
- 각 글은 기본적으로 `1차 생성 -> 2차 리뷰 -> 완료` 2단계를 거친다
