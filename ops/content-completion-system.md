# Content Completion System

## Goal

- 루틴 1세트는 중간 상태를 남기지 않고 `완료본`까지 가는 것을 기본으로 한다.
- 기본 흐름은 `1차 생성 -> 2차 리뷰 -> 완료` 2단계다.

## Default Rule

1. `1차 생성`
   - 키워드 확정
   - 제목, 설명, 본문 구조 작성
   - 내부 링크 기본 삽입
   - 이미지 슬롯 정의
   - 사진 넣기 전 상태까지 완성
2. `2차 리뷰`
   - 제목 강도 점검
   - 리드 보강
   - 실제 사례 또는 판단 기준 추가
   - 체크리스트, 비교표, 결정 규칙 같은 즉시 사용 요소 추가
   - 내부 링크 2개 이상 확인
   - 마지막 CTA 정리
   - `checklist/high-quality-post-checklist.md` 기준 통과 확인
3. `완료`
   - 이미지 반영
   - 실제 블로그 폴더로 이동
   - 배포 저장소 반영

## Quality Gate

- 아래 6개가 통과되지 않으면 완료로 보지 않는다.
  - 제목이 문제와 보상을 같이 말하는가
  - 리드가 독자의 실제 불편을 바로 잡는가
  - 본문에 실전 사례나 판단 기준이 있는가
  - 복사 가능한 체크리스트/결정 규칙/예시가 있는가
  - 내부 링크가 최소 2개 있는가
  - 마지막 문단이 다음 행동을 분명히 제시하는가
- 위 항목은 `docs/high-quality-post-direction.md`와 `checklist/high-quality-post-checklist.md`를 기준으로 더 엄격하게 본다.

## Exception Rule

- 기본은 2단계로 끝낸다.
- 아래 경우만 추가 수정 1회를 허용한다.
  - 구조가 틀린 경우
  - 리뷰와 글 매칭이 틀린 경우
  - 품질이 명백히 낮아 배포하기 어려운 경우

## State Rule

- 사용자 기준 상태는 3개만 쓴다.
  - `in_progress`
  - `image_needed`
  - `done`
- 내부적으로 더 많은 단계가 있더라도 사용자에게는 완료 중심으로만 보여준다.

## System Direction

- 사용자의 루틴을 여러 날로 쪼개도록 요구하지 않는다.
- 시스템이 내부 단계를 흡수하고, 한 번 시작한 세트는 가능한 범위 안에서 끝까지 밀어붙인다.
- 루틴의 기본 단위는 `초안`이 아니라 `완료본`이다.
