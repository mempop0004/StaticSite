# I18N Rule

- 서비스 목표 언어는 한국어와 영어 2개다.
- 현재 구현은 한국어 단일 구조이며, 새 기능은 나중에 한영 확장 가능성을 해치지 않아야 한다.
- 언어 구조를 도입할 때는 URL, 콘텐츠, 내비게이션을 함께 바꾼다.
- 한글과 영어는 단순 번역본이 아니라 같은 주제를 공유하는 대응 콘텐츠로 관리한다.
- 언어 구조 도입 전까지는 한국어 콘텐츠를 기준 원본으로 본다.
- 새 라우트나 콘텐츠 규칙을 만들 때는 언어 prefix 또는 언어별 루트 확장 가능성을 문서에 명시한다.
- 다국어 구조를 구현하기 전에는 언어별 파일을 임의로 섞어 만들지 않는다.
