# Content Parts Rule

이 프로젝트는 글 템플릿을 하나로 고정하지 않는다.

핵심 원칙은 `많은 부품을 준비하고, 글마다 가장 맞는 부품을 골라 조합하는 것`이다.

## Core Rule

- SEO 메타 태그는 선택이 아니라 필수다
- 글마다 구조가 달라도 된다
- 글은 `markdown` 중심으로 작성한다
- 필요할 때만 이미지를 붙인다
- 대표 이미지는 최소 `1개` 필요하다
- 본문 이미지는 필요할 때만 넣는다
- 텍스트로 충분하면 이미지보다 텍스트를 우선한다
- 설명 항목이 많으면 이미지보다 표를 우선한다
- 이미지가 필요해 보이면 먼저 제작을 시도한다
- 제작이 어려우면 이미지 슬롯으로 남기고 나중에 교체한다

## Quality Rule

- 좋은 글은 고정 템플릿이 아니라 좋은 부품 조합에서 나온다
- 부품은 최대한 많이 준비한다
- 한 글 안에서 너무 많은 목적을 섞지 않는다
- 각 섹션은 자기 역할이 분명해야 한다
- 장식용 섹션보다 정보 역할이 분명한 섹션을 우선한다

## Writing Rule

- 글을 쓰기 전에 먼저 글 목적과 검색 의도를 고른다
- 그 다음 섹션 부품을 고른다
- 필요하면 표, 체크리스트, 비교 블록, 이미지 슬롯을 붙인다
- 본문 품질은 글마다 최적화하되 템플릿 강제는 하지 않는다

## Block Rule

- 새 글을 쓸 때는 먼저 `docs/content-parts-system.md`에서 필드와 부품을 고른다
- 부품이 부족하면 새 부품을 추가하는 방향으로 본다
- 같은 부품은 여러 글에서 반복 사용 가능해야 한다
