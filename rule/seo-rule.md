# SEO Rule

이 문서는 블로그 운영 전에 확인해야 하는 최소 SEO 작성 기준을 정의한다.

## Purpose

- 검색 유입을 받을 수 있는 기본 구조를 유지한다
- 글마다 품질 편차가 너무 커지지 않게 한다
- 운영 속도를 해치지 않는 범위에서 반복 가능한 기준을 둔다

## Required Rule

- 모든 글은 `title`, `description`, `pubDate`, `blogId`, `stage`, `locale`를 가져야 한다
- 모든 글은 SEO 메타 태그를 필수로 가진다
- 모든 글은 canonical, open graph, twitter card 기준에서 빠지면 안 된다
- 모든 글은 대표 이미지가 있어야 한다
- 모든 글은 첫 문단에서 글의 문제와 범위를 바로 설명해야 한다
- 모든 글은 본문 안에 같은 블로그 단위의 내부 링크를 최소 `1개` 이상 포함해야 한다
- 모든 글은 `h2` 기준 섹션을 최소 `2개` 이상 가져야 한다
- 모든 글은 slug와 제목이 크게 어긋나면 안 된다

## Title Rule

- 제목은 검색 의도와 문제를 바로 드러내야 한다
- 제목은 가능하면 핵심 키워드를 앞쪽에 둔다
- 제목 길이는 UI 기준 `90자` 이내를 권장한다
- 과장형 문구, 낚시형 문구, 의미 없는 수식어를 넣지 않는다

## Description Rule

- description은 글 내용을 한두 문장으로 요약한다
- 제목을 그대로 반복하지 않는다
- 검색 결과에서 무엇을 해결하는 글인지 바로 이해돼야 한다
- description 길이는 UI 기준 `160자` 이내를 권장한다

## Meta Rule

- SEO 메타는 선택이 아니라 필수다
- 모든 글은 `title`, `description`, `canonical`, `og:title`, `og:description`, `og:image`, `twitter:card` 기준을 충족해야 한다
- locale 기준 alternate와 locale 메타는 페이지 라우트와 맞아야 한다
- 대표 이미지가 있더라도 메타 이미지로 연결되지 않으면 불완전한 상태로 본다

## Slug Rule

- slug는 짧고 명확하게 유지한다
- locale만 다르고 같은 주제의 글은 같은 slug를 유지한다
- 의미 없는 날짜, 내부 버전명, 임시 토큰을 넣지 않는다
- 검색 의도를 반영하는 핵심 단어를 우선 사용한다

## Body Rule

- 첫 문단에서 문제, 대상, 범위를 설명한다
- `h2`는 검색 사용자가 다시 찾을 만한 단위로 나눈다
- 한 섹션 안에서 너무 많은 주제를 섞지 않는다
- 의미 없는 긴 서론을 넣지 않는다
- 텍스트로 충분한 내용은 이미지보다 텍스트로 쓴다
- 텍스트 항목이 많으면 이미지보다 표를 우선한다

## Link Rule

- 같은 블로그 단위 안의 관련 글로 내부 링크를 연결한다
- 내부 링크 anchor text는 의미 있는 문장으로 쓴다
- "여기", "이 글" 같은 약한 링크 문구는 가급적 피한다
- 외부 링크는 꼭 필요한 경우만 쓴다

## Image Rule

- 대표 이미지는 필수다
- 이미지는 필요할 때만 넣는다
- 억지로 이미지를 넣지 않는다
- 이미지 안에 여러 줄 텍스트를 넣는 방식은 피한다
- 설명 항목이 많으면 이미지보다 표를 쓴다
- 내가 제작 가능한 이미지는 먼저 제작한다
- 제작이 어려운 경우는 이미지 슬롯으로 남기고 나중에 교체한다

## Alt Rule

- 대표 이미지와 본문 이미지는 alt를 가져야 한다
- 장식 이미지가 아니면 alt를 비워두지 않는다
- alt는 이미지 안의 정보나 역할을 짧게 설명한다
- 키워드를 억지로 반복하지 않는다

## Search Intent Rule

- 한 글은 하나의 핵심 검색 의도를 중심으로 쓴다
- 비교 글이면 비교 기준을 분명히 둔다
- 문제 해결 글이면 증상, 원인, 확인 순서를 명확히 둔다
- 운영 글이면 판단 기준과 실행 순서를 넣는다

## Publish Gate

- title이 문제를 설명하는가
- description이 글 범위를 설명하는가
- SEO 메타가 빠지지 않았는가
- 대표 이미지가 있는가
- 첫 문단이 문제를 바로 설명하는가
- `h2`가 최소 2개 이상 있는가
- 내부 링크가 최소 1개 이상 있는가
- slug가 제목과 맞는가

## Deferred Agreement

아래 항목은 나중에 합의해도 된다.

- 글 유형별 권장 단어 수
- 글 유형별 내부 링크 개수
- 대표 이미지 최소 해상도
- structured data 범위
- canonical 정책 세부 기준
