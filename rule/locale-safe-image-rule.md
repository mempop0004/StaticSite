# Locale-Safe Image Rule

- 기본 원칙: 이미지는 가능한 한 `언어 비의존`으로 만든다.
- 이미지 안에 한글/영문 문장을 넣지 않는다.
- 설명은 이미지 안이 아니라 본문, 캡션, 제목, 표에서 처리한다.
- 이미지는 `구조`, `흐름`, `비교`, `단계`, `객체 관계`를 보여주는 역할로 쓴다.
- 대표 이미지는 `즉시 인식형`이어야 한다.
- 썸네일을 보고 사용자가 해석해야 하면 실패로 본다.
- 대표 이미지는 `한 장면, 한 대비, 한 결과`만 남긴다.
- 본문 설명은 대표 이미지가 아니라 `inline` 이미지나 본문 텍스트가 맡는다.

## Good

- 아이콘 중심 구성
- 카드/패널 구조
- 화살표 흐름
- 숫자 단계 `1, 2, 3`
- 색 구분
- before / after
- 객체 관계
- 텍스트가 거의 없는 설명형 시각물
- 복잡함 하나와 정리된 결과 하나만 남긴 대표 커버

## Bad

- 이미지 안에 긴 한글 문장
- 이미지 안에 긴 영어 문장
- locale마다 새로 만들어야 하는 텍스트 설명 이미지
- 메모장, 채팅창, 자막처럼 텍스트가 핵심인 합성 이미지
- 여러 기능과 여러 과정을 한 장에 다 넣은 대표 이미지
- 인포그래픽처럼 읽어야 이해되는 대표 이미지

## Exception

- 실제 캡처가 필요한 `screenshot` 슬롯
- 서비스 UI 자체가 핵심인 경우
- 단, 이 경우도 텍스트는 최소화하고 꼭 필요한 장면만 쓴다.

## Role

- `cover`
  - 글 전체 개념을 언어 없이 한 장으로 요약
  - 설명형이 아니라 인상형이어야 한다
  - 썸네일에서 1초 안에 핵심 대비가 읽혀야 한다
- `inline`
  - 섹션의 흐름, 단계, 비교를 언어 없이 설명
- `comparison`
  - 두 대상 차이를 시각 구조로 보여줌
- `screenshot`
  - 실제 UI 위치 확인용. 예외적으로 언어 의존 가능
