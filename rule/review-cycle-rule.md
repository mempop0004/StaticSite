# Review Cycle Rule

- 기본 리뷰 사이클은 `글 작성 -> Gemini 리뷰 -> Codex 리뷰 -> 수정 -> 마감`이다.
- 리뷰는 파일로 남겨야 한다.
- 리뷰는 `이번 글 수정`과 `다음 글 규칙 업그레이드` 두 결과를 남겨야 한다.
- 일반 글은 기본 `1회 리뷰`로 끝낸다.
- 대표 글, pillar 글, 수익 연결 글만 `2회 리뷰`를 허용한다.
- 리뷰에서 나온 지적은 바로 믿지 말고 실제 파일 기준으로 사실 확인 후 반영한다.
- 모델 오판은 버리고, 반복적으로 맞는 지적만 규칙으로 승격한다.
- 최소 반영 우선순위는 `도입부`, `구체 예시`, `내부 링크`, `title/description`이다.
- 리뷰만 하고 끝내지 않는다. 다음 글에 반영할 규칙은 `review/memory/` 아래에 누적한다.
