# Current Image Decisions

## Goal

- 최근 반영한 이미지가 실제 블로그에 얼마나 맞는지 기록한다.
- 유지할 것과 다음에 교체하거나 개선할 것을 구분한다.

## Current Verdict

### blog_001 - 회의 메모를 할 일 목록으로 바꾸는 글

- 판정: `유지`
- 이유:
  - 글 주제가 `정리`, `구조화`, `할 일 목록화`라서 자료형 이미지와 잘 맞는다.
  - 설명 이미지도 본문 구조와 직접 연결된다.
  - 클릭 유도력은 더 올릴 수 있지만, 현재 반영분은 블로그용으로 충분히 성립한다.

### blog_002 - AI로 디버깅 가설을 빨리 좁히는 글

- 판정: `유지 가능, 재생성 후보`
- 이유:
  - 실제 반영은 가능하지만, 개발자 글 기준으로는 추상성이 남을 수 있다.
  - 개발자 문제 해결 글은 분위기보다 `증상`, `가설`, `검증` 구조가 먼저 보여야 한다.
  - 다음 비슷한 글부터는 같은 방향을 반복하지 않도록 규칙을 조정했다.

### blog_003 - sitemap could not be read 점검 글

- 판정: `유지`
- 이유:
  - 운영 문제 해결 글은 자료형 점검 구조와 가장 잘 맞는다.
  - 이번 이미지는 `live XML`, `응답`, `robots`, `host` 같은 글 핵심 구조와 연결된다.
  - 설명 이미지의 적합도가 높고, 블로그 본문과도 잘 맞는다.

## Summary

- `blog_001`: 유지
- `blog_002`: 유지 가능, 재생성 후보
- `blog_003`: 유지

## Next Decision Rule

- 앞으로 `개발자 글`에서 추상적 이미지가 나오면 우선 재생성 후보로 본다.
- `운영 글`은 자료형 다이어그램 구조가 맞으면 우선 유지한다.
- `일반 사용자 글`은 구조 변화가 보이면 유지 가능성이 높다.
