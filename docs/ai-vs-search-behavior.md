# AI Vs Search Behavior

이 문서는 사람들이 언제 AI에 묻고, 언제 검색을 하는지 구분해서 블로그 주제 축을 고르기 위한 기준 문서다.

## Core View

- 확정: AI와 검색은 같은 질문을 나눠 먹지 않는다
- 확정: 사람은 `바로 답이 필요할 때` AI에 묻는다
- 확정: 사람은 `근거, 비교, 선택, 탐색`이 필요할 때 검색한다
- 확정: 애드센스 중심 블로그는 검색에 남는 질문을 잡아야 한다

## When People Ask AI

- 답을 바로 요약받고 싶을 때
- 내 상황에 맞게 정리해 달라고 할 때
- 초안, 문구, 코드, 아이디어를 바로 만들고 싶을 때
- 여러 자료를 읽기 싫고 빠른 결론이 필요할 때
- 검색어를 잘 모르지만 질문 형태로 바로 묻고 싶을 때

## AI Friendly Query Types

- "이거 쉽게 설명해줘"
- "내 상황이면 뭐가 맞아?"
- "비교해서 결론만 말해줘"
- "초안 만들어줘"
- "이 에러 원인 추정해줘"

## When People Still Search

- 실제 사례를 보고 싶을 때
- 여러 선택지를 직접 비교하고 싶을 때
- 최신 정보와 원문 근거가 필요할 때
- 가격, 후기, 순위, 사용기, 캡처 화면이 필요할 때
- 특정 문제를 해결하기 위해 여러 페이지를 탐색해야 할 때
- 구매 전 마지막 검증이 필요할 때

## Search Friendly Query Types

- "`2026 ... 비교`"
- "`... 후기`"
- "`... 가격`"
- "`... 추천`"
- "`... 사용법`"
- "`... 해결`"
- "`... 설정`"
- "`... vs ...`"

## Monetization Meaning

- 확정: 애드센스는 `방문 수`만으로 끝나지 않는다
- 확정: 광고 수익에는 `검색 유입`, `페이지 깊이`, `지속 열람`이 중요하다
- 확정: 따라서 AI가 한 번에 끝내버리는 질문보다 검색과 탐색이 필요한 주제가 유리하다

## Topic Selection Rule

- 피해야 할 주제
- 한 문단 요약으로 끝나는 단순 설명형 주제
- AI가 일반론으로 바로 대답 가능한 주제
- 출처 검증이 거의 필요 없는 입문 정의형 주제

- 유리한 주제
- 비교가 필요한 주제
- 최신성 영향이 큰 주제
- 캡처, 절차, 경험, 설정값이 필요한 주제
- 선택 실수가 비용 손실로 이어지는 주제
- 한 글을 읽고 관련 글을 더 보게 되는 주제

## Good Categories For Adsense First

- 도구 비교
- 플랫폼 제한 비교
- 문제 해결 가이드
- 설정 체크리스트
- 실전 운영 사례
- 최신 정책 변화 정리

## Bad Categories For Adsense First

- 추상 개념 설명
- AI에게 바로 물으면 끝나는 요약형 지식
- 너무 개인화돼서 검색량이 작은 주제
- 검색 후 바로 이탈하는 일회성 짧은 답변형 주제

## Current Implication For This Project

- 확정: 이 프로젝트는 `비교형 + 문제 해결형 + 운영 실전형`을 모두 운영한다
- 확정: 초기 단계에서는 해보고 판단하는 것이 원칙이다
- 확정: 한 축만 먼저 고정하지 않고, 실험 후 취할 것과 버릴 것을 결정한다
- 가정: `블로그 수익화 운영론`만으로는 검색보다 AI에 흡수될 가능성이 높다
- 가정: `도구 비교`, `설정 오류 해결`, `정책 변화 대응`은 검색 잔존 수요가 크다

## Portfolio Rule

- 확정: 초기 운영은 `단일 주제 축 집중`이 아니라 `3축 병행 실험`이다
- 확정: 병행하는 축은 `기술 문제 해결형`, `툴 비교형`, `자동화 실전형`이다
- 확정: 평가 기준은 취향이 아니라 결과다
- 가정: 결과 기준은 검색 유입, 체류, 관련 글 이동, 광고 수익 기여다
