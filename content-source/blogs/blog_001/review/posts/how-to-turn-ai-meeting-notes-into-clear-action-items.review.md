# Review

- 회의가 끝난 뒤 할 일 목록이 안 남는 실제 불편에서 시작해 리드가 바로 문제를 잡는다.
- 본문이 `결정된 것 / 할 일 / 확인 필요 / 참고 메모`로 구조를 먼저 잡아 일반론으로 흐르지 않는다.
- `owner, action, due, blocker` 필드 기준을 줘서 바로 복사해 쓸 수 있는 판단 틀이 있다.
- 프롬프트 예시가 짧고 실전적이라 일반 사용자가 바로 적용하기 쉽다.
- 대표 이미지는 흩어진 메모가 구조화된 실행 목록으로 정리되는 방향이라 제목과 맞는다.

# Next

- 본문에 다른 글 안내 문장을 끼워 넣지 않는다.
- 내부 링크가 필요하면 관련 글 블록이나 마지막 1문장 수준으로만 제한한다.
- 회의 메모 글은 항상 `누가 / 무엇을 / 언제까지`가 본문 중심에 남아야 한다.
