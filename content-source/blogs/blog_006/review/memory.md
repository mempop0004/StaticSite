# Review Memory

- SEO 글은 추상 원칙보다 실제 점검 항목을 우선한다.
- 메타 설명은 결과와 실수를 같이 담는다.
- Search Console 맥락은 스크린샷 없이도 이해되게 쓴다.
- 검색 유입 글은 인덱싱, 메타, 내부 링크, 클릭률 포인트를 분리해서 설명한다.
- 문제 해결형 글은 무엇부터 확인할지 순서를 명확히 준다.
- 기준 글은 유닛 페이지 링크와 최소 한 개의 실행 예시를 같이 넣는다.
- 검색 운영 이미지는 추상 신호 흐름보다 `검색 결과 기대`, `클릭 전 약속`, `랜딩 확인 장면`을 우선한다.
- 검색 운영 hero에 Google 로고나 상태 라벨 텍스트를 직접 넣으면 설명보다 스크린샷 포스터처럼 보여 기준이 약해진다.
- `Crawled - currently not indexed`류 이미지는 로고/문구보다 `도달은 했지만 보류된 상태`를 객체와 명암으로 보여주는 편이 낫다.
