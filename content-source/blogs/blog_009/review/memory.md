# Review Memory

- 부품 설명은 실제 작성 예시와 같이 둔다.
- 품질 규칙은 체크리스트로 끝내지 말고 적용 예시를 같이 쓴다.
- 이미지 역할은 장식보다 설명 우선으로 본다.
- 기준 글은 유닛 페이지 링크와 발행 단계 예시를 같이 남긴다.
- content system용 001은 난잡한 종이 소용돌이, 흩어진 조각, clutter cloud로 가지 않게 막는다.
- content system용 001은 정돈된 board, before/after split, single handoff scene처럼 차분한 구조로 간다.
- 002는 카드형 단계 보드가 더 적합했고, 글 설명에 직접 붙는 자료형 구조를 우선한다.
- 메타 시스템 이미지도 역할은 맞았지만 `TITLE`, `SUMMARY`, `TAGS` 같은 직접 라벨이 들어가면 현재 기준으로는 미달이다.
- 메타 이미지는 제목/설명/태그를 글자 대신 카드 구조와 연결 관계만으로 보여줘야 한다.
