# Review

- 글은 증상과 원인을 먼저 분리해서, `스타일이 깨져 보인다`와 `자산 경로가 어긋났다`를 다른 문제로 읽게 만든다.
- 2번 섹션이 가장 강하게 서 있어서 페이지 경로와 자산 경로가 서로 다를 수 있다는 핵심 진단 축이 포스팅다운 밀도로 설명된다.
- `공개 페이지 URL / CSS 요청 / JS 요청 / base path`를 비교하는 기준이 남아서 바로 적용 가능한 점검 순서가 있다.
- `002` 이미지는 경로 분기와 자산 요청 구조를 설명하는 용도로 적합하다.
- `001`은 깨진 상태와 복구 상태의 대비라는 방향은 맞지만, UI 카드와 버튼 텍스트가 많아 hero가 약간 광고형/목업형으로 보인다. 다음 hero는 더 적은 요소, 더 직접적인 broken vs fixed 장면으로 가는 편이 낫다.
