# Review

- 글의 중심을 canonical 단독 점검이 아니라 `실제 페이지 URL / canonical 대상 / redirect 동작` 세 신호를 같이 비교하는 데 두면서 실전 문제 해결 글다운 판단 기준이 분명해졌다.
- 2번 섹션이 가장 길고 강해서, 왜 이 세 신호를 한 번에 봐야 하고 서로 다른 정체성 신호가 어떤 식으로 충돌하는지가 포스팅다운 밀도로 전개된다.
- `템플릿을 고치기 전 먼저 볼 세 가지`와 `고친 뒤 재발 방지`가 함께 남아서 바로 적용 가능한 점검 순서가 있다.

- 다만 이번 이미지는 `locale URL`, `comparison node`, `Result 1` 같은 직접 텍스트가 많아서 현재 기준으로는 미달이다. 다음부터는 신호 아이콘과 경로만으로 구조를 읽히게 해야 한다.
