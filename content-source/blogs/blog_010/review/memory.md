# Review Memory

- 증상과 원인을 분리해서 쓴다.
- 독자가 가장 먼저 확인할 3가지를 먼저 준다.
- 해결 이후 재발 방지 항목을 같이 적는다.
- 경로 규칙과 redirect 예시는 실제 점검 순서로 써서 바로 적용되게 한다.
- hero 이미지는 broken vs fixed 대비는 좋지만, 카드/버튼/UI 텍스트가 많아지면 광고형 mockup처럼 보이므로 더 직접적이고 요소 수가 적은 장면으로 간다.
- inline 이미지는 경로 분기, 요청 흐름, 자산 위치처럼 실제 점검 구조가 보이는 자료형이 더 적합하다.
- locale/canonical 이미지는 구조 방향은 맞았지만 `locale URL`, `comparison node`, `Result 1` 같은 직접 텍스트가 들어가면 품질이 급격히 떨어진다.
- URL 신호 충돌 이미지는 페이지 URL, canonical, redirect 세 신호를 아이콘과 경로만으로 구분해야 하고, 텍스트 라벨에 기대면 안 된다.
