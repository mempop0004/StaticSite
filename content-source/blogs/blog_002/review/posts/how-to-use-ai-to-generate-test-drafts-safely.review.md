# Review

- 글 중심을 `AI가 테스트를 대신 쓴다`에서 `빈 화면 비용만 줄이고 판단은 사람이 끝까지 가진다`로 바꿨다.
- 출발점을 `잘 맞는 모듈`과 `위험한 모듈`로 나눠 일반론을 줄였다.
- `normal path / failure path / edge case` 구조로 요청하는 방식을 넣어 바로 복사 가능한 흐름으로 정리했다.
- assertion, 숨은 가정, invented setup 검토 항목을 분리해 실전 체크리스트 성격을 강화했다.
- 대표 이미지는 실사형 IDE 장면이 아니라 `거의 완성된 테스트 초안 + 아직 남은 판단`을 보여주는 flat cover 방향으로 고정한다.
